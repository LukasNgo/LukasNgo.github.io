Room Access Program written using Object Oriented Design Patterns

This project is a Java program with GUI which has been designed and developed using NetBeans IDE and coded using the State and Observer object oriented design patterns. It is a software system for controlling access to rooms on the university campus. The State pattern is used to implement two emergency states for each room, the observer pattern is used to create new persons, rooms and buildings. I created UML class diagrams which can be seen below. The program is using abstract classes which other classes inherit from. The program also contains a full Javadoc documentation.

Website with server-side technology

The project is a parallax scrolling portfolio template website with a snake minigame and a chat application. The main idea behind this project was to create a template for a portfolio website for games development students.

The website contains 3 pages, main page - portfolio is a parallax scrolling website made using HTML and CSS with images and different sections that people can use to show off their work.

Second page - Minigames is a simple snake game written using JavaScript that people can play and compete against each other online. The game scores and names are saved into a server-side database using Socket.io and MongoDB.

Last page - Chat is a simple chat application that lets people communicate with each other or leave comments. It's written using JavaScript and is using Socket.io and MongoDB technology.

Technologies included are:

Node.js

Express

Socket.io

MongoDB

Project requires Node.js and MongoDB installed to run.
